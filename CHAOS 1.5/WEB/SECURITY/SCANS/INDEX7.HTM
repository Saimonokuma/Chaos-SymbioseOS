    The test conditions include:
  • The device under test (DUT) consists of two nodes in a single active cluster
  • At least one testapp process executes migrated for the duration of the test
  • The DUT and the Nessus server are on the same LAN segment
  • Nessus is configured with ALL plugins enabled
  • Nessus is configured not to ping (TCP/ICMP), to prevent DUTs from deceiving the scanner
  • Nessus is configured for SYN type TCP scanning, in an aggressive scan mode
  • The assessment date is the date that the Nessus plugins were updated (immediately prior to the test)
  • Tests that show obvious false positives (such as a phantom 514/UDP syslog service) are reconducted
    The values [x/y/z] reflect Nessus' x holes, y warnings and z notes, respectively.



What is Nessus?


    The Nessus home page describes its purpose, best;

        The "Nessus" Project aims to provide to the internet community a free, powerful, up-to-date
        and easy to use remote security scanner. A security scanner is a software which will audit
        remotely a given network and determine whether bad guys (aka 'crackers') may break into it,
        or misuse it in some way.



So, how secure is CHAOS?


    In production deployment models, CHAOS greatly reduces risk in two key ways; minimisation of the operating software environment, and hardening of the resultant, minimised, operating software environment

    The luxury offered by the Single System Image architecture is that only one node is required to contain the production user-space software environment. By minimising the operating environment deployed to the rest of your production cluster (the vast majority of it), you can effectively reduce your software install base from (for example) 650Mbytes to (CHAOS) 6Mbytes; potentially reducing your risk by a proportionate amount. By hardening the remaining 6Mbyte install base, the final risk exposure is greatly reduced - significantly lower than the default installation of any compatible environment.

    Unless your network consists solely of CHAOS nodes; there will be far easier things to hack on your network, than a CHAOS cluster.
